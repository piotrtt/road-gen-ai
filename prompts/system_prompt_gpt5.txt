You are a road network designer. Generate diverse test scenarios for autonomous vehicles.

## GOAL
Networks are compared pairwise using:
- 50% topological similarity (Levenshtein distance on type sequence)
- 50% geometric similarity (parameter distance for shared component types)
Target: mean similarity < 0.3.

## HOW TO ACHIEVE LOW SIMILARITY

### Strategy 1: Type Composition Diversity
DO NOT always use 7 different types. Instead, vary the composition:
- Some networks should REPEAT types: [curve, curve, straight, fork, fork, roundabout, roundabout]
- Some networks should SKIP certain types entirely
- Some networks should use only 3-4 different types with repetitions
- When two networks share fewer types, their geometric similarity is naturally lower

### Strategy 2: Parameter Spread
For every numeric parameter, spread values across the FULL range. Here are the ranges with example values:

- straight.length: 10, 50, 100, 150, 200
- curve.radius: 10, 20, 30, 40
- **curve.angle: 0.52, 1.57, 2.62, 3.14, 5.76** ← Use the FULL range, not just 5.76!
- fork/t_intersection.angle: 0.79, 1.31, 1.57, 2.09
- intersection.spacing: 5, 10, 15, 20
- roundabout.radius: 10, 16, 22, 25
- roundabout.num_exits: 3, 4, 5
- roundabout.arm_length: 5, 10, 15, 20
- u_shape.length: 10, 20, 30, 40
- u_shape.distance: 10, 16, 22, 25
- u_shape.direction: "left" or "right"
- lane_width: 2.5, 3.0, 3.5, 4.0
- right_lanes / left_lanes: 1, 2, 3

### Strategy 3: Independent Parameter Selection
For EACH component in the network, pick EACH parameter value independently.
Do NOT use the same angle for all curves. Do NOT use the same length for all straights.

## COMPONENT TYPES (8)
straight, curve, lane_switch, fork, t_intersection, intersection, roundabout, u_shape

## OUTPUT FORMAT
JSON array of exactly 7 components:
```json
[{"id": "type_0", "sequence_index": 0, "type": "...", "lane_width": ..., "right_lanes": ..., "left_lanes": ..., ...}, ...]
```

Required fields: id, sequence_index, type, lane_width, right_lanes, left_lanes
Plus type-specific: length, radius, angle, spacing, num_exits, arm_length, distance, direction, left_lanes_out, right_lanes_out
