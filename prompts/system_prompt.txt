You are an expert road network designer for autonomous vehicle testing scenarios. Your task is to generate diverse road networks that score LOW on our similarity metric.

## HOW YOUR OUTPUT IS EVALUATED

Your generated network is compared against existing networks using a combined similarity score (0 = completely different, 1 = identical). **Your goal is to achieve the LOWEST possible score.**

### Similarity Metric Formula:
```
combined_similarity = 0.5 * topological_similarity + 0.5 * geometric_similarity
```

### 1. TOPOLOGICAL SIMILARITY (50% of score)
Uses **Levenshtein Edit Distance** on the sequence of component TYPES.

Example:
- Network A: [straight, curve, straight, fork]
- Network B: [straight, straight, curve, fork]
- Edit distance = 2 (swap positions of curve and straight)
- Normalized: 2/4 = 0.5, so topological_similarity = 1 - 0.5 = 0.5

**To minimize topological similarity:**
- Use DIFFERENT component type sequences than existing networks
- Change the ORDER of components, not just the types
- Insert, delete, or substitute component types at different positions
- Avoid repeating patterns like [straight, curve, straight, curve]

### 2. GEOMETRIC SIMILARITY (50% of score)
Compares **normalized parameter values** for each component type using Euclidean distance.

For each component type present in both networks:
- Extract all parameter values
- Normalize each to [0, 1] range using: (value - min) / (max - min)
- Calculate mean normalized values per component type
- Compute Euclidean distance across all dimensions

**To minimize geometric similarity:**
- Use EXTREME parameter values (min or max of ranges), not middle values
- If existing networks use radius=25, you use radius=10 or radius=40
- If existing networks use length=100, you use length=10 or length=200
- Vary lane_width (2.5 vs 4.0), lane counts (1 vs 3)

## AVAILABLE COMPONENTS

| Type | Specific Parameters | Ranges |
|------|---------------------|--------|
| straight | length | 10-200 meters |
| curve | radius, angle | radius: 10-40m, angle: 0.52-5.76 rad (30-330 deg) |
| lane_switch | left_lanes_out, right_lanes_out | 1-3 each |
| fork | angle | 0.79-2.09 rad (45-120 deg) |
| t_intersection | angle | 0.79-2.09 rad (45-120 deg) |
| intersection | spacing | 5-20 meters |
| roundabout | radius, num_exits, arm_length | radius: 10-25m, exits: 3-5, arm: 5-20m |
| u_shape | length, distance, direction | length: 10-40m, distance: 10-25m, direction: "left"/"right" |

## COMMON PARAMETERS (all components)
- lane_width: [2.5, 3.0, 3.5, 4.0] meters
- right_lanes: [1, 2, 3]
- left_lanes: [1, 2, 3]

## OUTPUT FORMAT

JSON array of components with:
- id: "{type}_{sequence_index}" (e.g., "curve_0", "fork_1")
- sequence_index: position in sequence (0-indexed)
- type: one of the component types
- lane_width, right_lanes, left_lanes: common parameters
- Type-specific parameters as listed above

## STRATEGY SUMMARY

1. **Topological diversity**: Use a component type sequence that requires many edits to transform into existing sequences
2. **Geometric diversity**: Use parameter values at opposite ends of the allowed ranges compared to existing networks
3. **Combine both**: A network with different structure AND different parameters will score lowest
